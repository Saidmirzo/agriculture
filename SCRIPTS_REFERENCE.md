# 📋 Deployment Scripts Reference

## Scripts Overview

Sizning Django backend-ni Ubuntu serverga deploy qilish uchun barcha scripts:

| # | Script | Maqsad | Tavsif |
|---|--------|--------|--------|
| 1 | `deploy.sh` | Initial Setup | Barcha packages, database, services |
| 2 | `setup-ssl.sh` | SSL Certificate | HTTPS + Let's Encrypt |
| 3 | `update-deploy.sh` | Production Update | Git pull + migrations + restart |
| 4 | `manage-services.sh` | Service Management | Interactive menu (13 options) |
| 5 | `backup.sh` | Backup/Restore | Database va files backup |

---

## 1. DEPLOY.SH - INITIAL SETUP

**Birinchi marta ishga tushirish uchun. Barcha services automatik o'rnatiladi.**

```bash
sudo ./deploy.sh
```

### O'rnatiladi:
```
✅ System packages (nginx, postgresql, redis, python3)
✅ PostgreSQL database va user
✅ Redis server
✅ Python 3.10+ virtual environment
✅ Django packages (requirements.txt dan)
✅ Gunicorn HTTP API server (port 8000)
✅ Daphne WebSocket/ASGI server (port 8001)
✅ Nginx reverse proxy (ports 80, 443)
✅ Systemd services (auto-start, auto-restart)
✅ Firewall configuration (UFW)
✅ Self-signed SSL certificate
✅ Application directory (/var/www/agriculture)
✅ Database backup (credentials faylda)
```

### Chiqarish:
```
📁 /var/www/agriculture/          - Application directory
📁 /var/www/agriculture/venv/      - Virtual environment
📄 /var/www/agriculture/.env       - Environment variables
📄 /var/www/agriculture/DATABASE_CREDENTIALS.txt - Credentials
🔧 /etc/systemd/system/gunicorn_agriculture.service
🔧 /etc/systemd/system/daphne_agriculture.service
🔧 /etc/nginx/sites-available/agriculture
```

### Zaruriy o'zgaruvchanlar (Script ichida):
```bash
DOMAIN="your-domain.com"           # O'zgarish kerak!
EMAIL="your-email@example.com"     # O'zgarish kerak!
DB_PASSWORD=<auto-generated>       # Random password
APP_DIR="/var/www/agriculture"
VENV_DIR="/var/www/agriculture/venv"
```

### Davomiyligi: 15-20 minut

## 2. SETUP-SSL.SH - HTTPS CERTIFICATE

**Let's Encrypt orqali free SSL sertifikat o'rnatish. HTTPS faolashtiriladi.**

```bash
sudo ./setup-ssl.sh
```

### Input:
```
Sorov 1: Domain nomi?        (masalan: agriculture.uz)
Sorov 2: Email address?      (masalan: admin@agriculture.uz)
```

### Avtomatik qiladi:
```
✅ Certbot o'rnatish (agar kerak)
✅ SSL sertifikat olish
✅ Nginx konfiguratsiyasini yangilash
✅ HTTP → HTTPS redirect
✅ Auto-renewal cron job qo'shish (har 90 kundа)
✅ Daphne WebSocket SSL sozlash
```

### Chiqarish:
```
📄 /etc/letsencrypt/live/domain/fullchain.pem - Public certificate
📄 /etc/letsencrypt/live/domain/privkey.pem   - Private key
🔄 Auto-renewal: /usr/bin/certbot renew       - Automatic (3 AM daily)
```

### Davomiyligi: 5-10 minut

## 3. UPDATE-DEPLOY.SH - PRODUCTION UPDATE

**Production-da yangi kod deploy qilish uchun. Xavfli emas (health check bor).**

```bash
sudo ./update-deploy.sh
```

### Qiladi (automatic):
```
[1/6] Git repository update        - git pull origin main
[2/6] Python packages update       - pip install -r requirements.txt
[3/6] Django migrations            - python manage.py migrate --noinput
[4/6] Collect static files         - python manage.py collectstatic --noinput
[5/6] Restart services             - systemctl restart
[6/6] Health check                 - Services running?
```

### Ishga tushirish:
```bash
cd /var/www/agriculture
sudo ./update-deploy.sh
```

### Davomiyligi: 2-5 minut

## 4. MANAGE-SERVICES.SH - SERVICE MANAGEMENT MENU

**Interactive menu. Services boshqarish, logs ko'rish, backup qilish.**

```bash
./manage-services.sh
```

### Menyu Options (13 ta):

```
1. Services status ko'rish
   └─ Gunicorn, Daphne, Nginx, PostgreSQL, Redis

2. Services qayta ishga tushirish
   └─ sudo systemctl restart

3. Git pull (code update)
   └─ git pull origin main

4. Django migrations qilish
   └─ python manage.py migrate

5. Static files collect qilish
   └─ python manage.py collectstatic

6. Logs ko'rish (Gunicorn)
   └─ sudo journalctl -u gunicorn_agriculture

7. Logs ko'rish (Daphne)
   └─ sudo journalctl -u daphne_agriculture

8. Logs ko'rish (Nginx)
   └─ sudo journalctl -u nginx

9. Redis test qilish
   └─ redis-cli ping

10. Database backup qilish
    └─ pg_dump agriculture_db > backup_DATE.sql

11. Services stop qilish
    └─ systemctl stop

12. Services start qilish
    └─ systemctl start

13. SSH key qo'shish
    └─ Add to ~/.ssh/authorized_keys
```

### Ishga tushirish:
```bash
./manage-services.sh
# Keyin 1-13 oralig'ida raqam kiriting
```

## 5. BACKUP.SH - BACKUP VA RESTORE

**Database va files backup qilish, keyin restore qilish.**

```bash
sudo ./backup.sh <command> [options]
```

### Commands:

#### BACKUP - Full backup qilish
```bash
sudo ./backup.sh backup
```
Backups:
- ✅ PostgreSQL database (`db_backup_DATE.sql.gz`)
- ✅ Media files (`media_backup_DATE.tar.gz`)
- ✅ Static files (`static_backup_DATE.tar.gz`)
- ✅ Code (`code_backup_DATE.tar.gz`)

Joylash: `/var/backups/agriculture/`

#### LIST - Backups ro'yxati
```bash
sudo ./backup.sh list
```
Barcha backuplarni ko'rish

#### RESTORE DATABASE
```bash
sudo ./backup.sh restore-db db_backup_20240607_120000.sql.gz
```
Parametr: backup fayli nomi

#### RESTORE FILES
```bash
sudo ./backup.sh restore-files media media_backup_20240607_120000.tar.gz
```
Parametrlar:
- Type: `media`, `static`, yoki `code`
- Backup file nomi

#### CLEANUP - Old backups o'chirish
```bash
sudo ./backup.sh cleanup
```
30 kundan eski backuplarni o'chiradi

### Backup Location
```
/var/backups/agriculture/
├── db_backup_20240607_120000.sql.gz
├── media_backup_20240607_120000.tar.gz
├── static_backup_20240607_120000.tar.gz
└── code_backup_20240607_120000.tar.gz
```

## 📊 Service Architecture

```
REQUEST FLOW:
User → HTTPS (443) → Nginx
                        ├─ /ws/ → Daphne (8001) WebSocket
                        ├─ /api/ → Gunicorn (8000) HTTP
                        └─ /static/ → Static Files

BACKEND SERVICES:
┌─────────────────────────────────────────┐
│ Gunicorn (HTTP API)    port 8000        │
│ Daphne (WebSocket)     port 8001        │
│ PostgreSQL (Database)  port 5432        │
│ Redis (Cache)          port 6379        │
│ Nginx (Proxy)          port 80, 443     │
└─────────────────────────────────────────┘
```

## 🔧 Common Tasks

### Restart specific service
```bash
sudo systemctl restart gunicorn_agriculture
sudo systemctl restart daphne_agriculture
sudo systemctl restart nginx
```

### View logs real-time
```bash
sudo journalctl -u gunicorn_agriculture -f
sudo journalctl -u daphne_agriculture -f
sudo journalctl -u nginx -f
```

### SSH into database
```bash
sudo -u postgres psql agriculture_db
```

### Manual Django shell
```bash
cd /var/www/agriculture
source venv/bin/activate
python manage.py shell
```

### Test Redis
```bash
redis-cli ping
redis-cli INFO memory
```

### Check disk space
```bash
df -h
du -sh /var/www/agriculture
```

## 🆘 Quick Troubleshooting

| Problem | Solution |
|---------|----------|
| 502 Bad Gateway | `sudo systemctl restart gunicorn_agriculture daphne_agriculture` |
| Database error | `sudo systemctl status postgresql` |
| Redis error | `redis-cli ping` |
| Nginx error | `sudo nginx -t` then `sudo systemctl restart nginx` |
| Service won't start | `sudo journalctl -u gunicorn_agriculture -f` |
